# React Chat App (`apps/chat`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

> **Revised after the live test (2026-06-21).** Tasks 3 and 5 below describe the
> original "direct from browser via `VITE_FLUE_RUNNER_URL`" approach. That can't
> work — the Flue runtime serves no CORS and rejects `OPTIONS` preflight — so the
> shipped code instead proxies through a same-origin server entry
> (`src/server.ts`) to a **server-side** `FLUE_RUNNER_URL`, passes a bound
> `fetch` to the SDK, and keeps a client-side transcript (the runtime doesn't
> echo user messages). See the design spec's "Connection" and "Implementation
> findings" sections for the authoritative current design; treat the task code
> blocks below as historical.

**Goal:** Add `apps/chat`, a TanStack Start (React) chat interface that talks to the `d0lt-bot` Flue agent, with the runner endpoint configured via an env var.

**Architecture:** A new workspace app scaffolded with the official `@tanstack/cli create` (React framework, Tailwind always-on, no examples). The browser connects directly to the Flue runner via `createFlueClient({ baseUrl: VITE_FLUE_RUNNER_URL })`, wrapped in `FlueProvider`. A single chat route uses `useFlueAgent({ name: 'd0lt-bot', id })` where `id` is a conversation id persisted in `localStorage`. Pure helpers (env guard, conversation id) are unit-tested with Vitest; the UI stays thin and untested.

**Tech Stack:** TanStack Start (`@tanstack/react-start`, `@tanstack/react-router`), React 19, Vite, Tailwind CSS, `@flue/react` + `@flue/sdk` (`1.0.0-beta.1`), Vitest 4, TypeScript ESM, pnpm workspace.

## Global Constraints

- Package manager is **pnpm 11** (`packageManager: pnpm@11.8.0`). Never use npm/yarn to install into the workspace. (The one exception: the `@tanstack/cli` scaffolder is invoked via `pnpm dlx`.)
- Node **24.x** (`engines.node`).
- App name in `apps/chat/package.json` must be **`chat`** (so `pnpm --filter chat ...` works).
- Use the existing pnpm **catalogs** for shared dev deps where they exist: `vitest: catalog:testing`, `@types/node: catalog:tooling`, `typescript: catalog:tooling`.
- Flue packages pinned to **`1.0.0-beta.1`** (matches the `@flue/*` versions already in the repo).
- The agent name is the constant **`d0lt-bot`**. Only the endpoint is env-driven (`VITE_FLUE_RUNNER_URL`).
- TypeScript ESM, `NodeNext`-style. Tests are pure and offline — no network, no live Flue runtime.
- Lint/format is oxlint + oxfmt at the repo root; run `pnpm lint` before final commit.

```ts
import { createFlueClient } from "@flue/sdk";

/**
 * Build the Flue SDK client that the browser uses to reach the runner.
 *
 * `baseUrl` comes from `import.meta.env.VITE_FLUE_RUNNER_URL` at the call site;
 * it is passed in (rather than read here) so this guard is unit-testable.
 */
export function createChatClient(baseUrl: string | undefined) {
  if (!baseUrl) {
    throw new Error(
      "VITE_FLUE_RUNNER_URL is not set. Point it at your Flue runner, e.g. http://localhost:3583",
    );
  }
  return createFlueClient({ baseUrl });
}
```

### Task 4: Conversation id persistence helper (TDD)

**Files:**
- Create: `apps/chat/src/lib/conversation.ts`
- Test: `apps/chat/src/lib/conversation.test.ts`

**Interfaces:**
- Produces:

- [ ] **Step 1: Write the failing test**

Create `apps/chat/src/lib/conversation.test.ts`:

```ts
import { describe, expect, it } from "vitest";
import { getOrCreateConversationId } from "./conversation.ts";

function fakeStorage(initial: Record<string, string> = {}) {
  const map = new Map(Object.entries(initial));
  return {
    getItem: (k: string) => map.get(k) ?? null,
    setItem: (k: string, v: string) => {
      map.set(k, v);
    },
    dump: () => Object.fromEntries(map),
  };
}

describe("getOrCreateConversationId", () => {
  it("generates, stores, and returns a new id when none exists", () => {
    const storage = fakeStorage();
    const id = getOrCreateConversationId(storage, () => "generated-id");
    expect(id).toBe("generated-id");
    expect(storage.dump()["d0lt-chat:conversation-id"]).toBe("generated-id");
  });

  it("returns the existing id on subsequent calls", () => {
    const storage = fakeStorage({ "d0lt-chat:conversation-id": "existing-id" });
    const id = getOrCreateConversationId(storage, () => "should-not-be-used");
    expect(id).toBe("existing-id");
  });
});
```


- [ ] **Step 3: Write the minimal implementation**

Create `apps/chat/src/lib/conversation.ts`:

```ts
const STORAGE_KEY = "d0lt-chat:conversation-id";

/**
 * Get the persisted conversation id, or create + persist a new one. `storage`
 * is injected so this is testable without a real `localStorage`.
 */
export function getOrCreateConversationId(
  storage: Pick<Storage, "getItem" | "setItem">,
  makeId: () => string = () => crypto.randomUUID(),
): string {
  const existing = storage.getItem(STORAGE_KEY);
  if (existing) return existing;
  const id = makeId();
  storage.setItem(STORAGE_KEY, id);
  return id;
}
```

- [ ] **Step 2: Build the Chat component**

Create `apps/chat/src/components/Chat.tsx`:

```tsx
import { useFlueAgent } from "@flue/react";
import { useState } from "react";
import { getOrCreateConversationId } from "../lib/conversation.ts";

const AGENT_NAME = "d0lt-bot";

export function Chat() {
  const [conversationId] = useState(() =>
    getOrCreateConversationId(window.localStorage),
  );
  const [input, setInput] = useState("");
  const [sending, setSending] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const agent = useFlueAgent({ name: AGENT_NAME, id: conversationId });

  async function submit(event: React.FormEvent) {
    event.preventDefault();
    const text = input.trim();
    if (!text || sending) return;
    setInput("");
    setSending(true);
    setError(null);
    try {
      await agent.sendMessage(text);
    } catch (err) {
      setError(err instanceof Error ? err.message : "Failed to reach the runner.");
    } finally {
      setSending(false);
    }
  }

  return (
    <main className="mx-auto flex h-screen max-w-2xl flex-col">
      <header className="border-b px-4 py-3 text-sm font-semibold text-gray-700">
        d0lt-bot
      </header>

      <div className="flex-1 space-y-3 overflow-y-auto px-4 py-4">
        {agent.messages.length === 0 && (
          <p className="text-center text-sm text-gray-400">
            Ask d0lt-bot to review a PR or run a repo's tests.
          </p>
        )}
        {agent.messages.map((message) => (
          <div
            key={message.id}
            className={message.role === "user" ? "text-right" : "text-left"}
          >
            <span className="mb-1 block text-xs uppercase tracking-wide text-gray-400">
              {message.role}
            </span>
            <div
              className={
                "inline-block whitespace-pre-wrap rounded-lg px-3 py-2 text-sm " +
                (message.role === "user"
                  ? "bg-blue-600 text-white"
                  : "bg-gray-100 text-gray-900")
              }
            >
              {message.parts.map((part, i) =>
                part.type === "text" ? <span key={i}>{part.text}</span> : null,
              )}
            </div>
          </div>
        ))}
        {sending && (
          <p className="text-left text-sm text-gray-400">d0lt-bot is thinking…</p>
        )}
        {error && <p className="text-left text-sm text-red-600">{error}</p>}
      </div>

      <form onSubmit={submit} className="flex gap-2 border-t px-4 py-3">
        <input
          className="flex-1 rounded-md border px-3 py-2 text-sm focus:outline-none focus:ring"
          value={input}
          placeholder="Message d0lt-bot…"
          onChange={(e) => setInput(e.target.value)}
        />
        <button
          type="submit"
          disabled={!input.trim() || sending}
          className="rounded-md bg-blue-600 px-4 py-2 text-sm font-medium text-white disabled:opacity-50"
        >
          Send
        </button>
      </form>
    </main>
  );
}
```

> If `message.parts[].type` or the `useFlueAgent` return shape differs from the guide, adjust the rendering to match `@flue/react`'s actual types (the typecheck in Step 4 will surface mismatches). The guide example is the reference: `agent.messages`, `agent.sendMessage`, `message.role`, `message.parts` with `part.type === "text"` and `part.text`.
